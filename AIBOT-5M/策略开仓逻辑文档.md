## 一、策略架构总览

### 1.1 双引擎架构
```
┌─────────────────────────────────────────────────────┐
│           决策引擎 (DecisionEngine)                  │
├─────────────────────────────────────────────────────┤
│  15m 市场状态识别   →   5m 执行打分                   │
│  (TREND/RANGE)       (趋势跟随/区间回归)               │
└─────────────────────────────────────────────────────┘
```

### 1.2 市场状态分类
| 状态 | 判定条件 | 交易模式 |
|------|---------|---------|
| TREND | ADX >= 15.0 | 趋势跟随 |
| RANGE | ADX <= 14.5 | 区间回归 |
| NO_TRADE | 14.5 < ADX < 15.0 | 暂停交易 (除非激活趋势捕获) |

## 三、开仓决策流程

### 3.1 决策主流程
```
1. 市场状态检测 (_detect_regime)
   └─ 判断 TREND / RANGE / NO_TRADE

2. 获取引擎参数 (_engine_params_for)
   └─ 根据市场状态加载对应门槛和参数

3. 计算动态权重 (DeepSeekWeightRouter)
   └─ AI优化 + 本地规则回退

4. 计算15m主资金分数
   └─ _score_with_weights (TREND) 或 _score_range (RANGE)

5. 计算5m执行分数
   └─ _score_with_weights (TREND) 或 _score_range (RANGE)

6. 融合15m + 5m分数 (_fuse_scores)
   └─ 加权融合 + 一致性增强

7. 比较开仓门槛
   └─ long_score >= long_threshold OR short_score >= short_threshold

8. 执行开仓决策
   └─ 返回 FundFlowDecision (BUY/SELL/HOLD)
```

### 3.2 关键判定条件
- **ATR波动率过滤**: `atr_pct >= 0.0008 AND atr_pct <= 0.02`
- **极端波动冷却**: 连续2根K线ATR>0.02时冷却30分钟

## 四、开仓评分计算公式

### 4.1 TREND 模式评分公式

#### 4.1.1 做多评分 (long_score)
```
long_score = (
    0.24 × max(cvd_ratio, 0)           // CVD累计偏差
  + 0.14 × max(cvd_momentum, 0)        // CVD动量
  + 0.22 × max(oi_delta_ratio, 0)      // 持仓量增量
  + 0.10 × max(-funding_rate, 0)       // 负资金费率看多
  + 0.15 × max(depth_ratio - 1.0, 0)    // 深度比率看多
  + 0.15 × max(imbalance, 0)           // 买卖盘不平衡
  + liquidity_norm_factor × max(liquidity_delta_norm, 0)  // 流动性增量
)
```

#### 4.1.2 做空评分 (short_score)
```
short_score = (
    0.24 × max(-cvd_ratio, 0)          // CVD累计偏差
  + 0.14 × max(-cvd_momentum, 0)       // CVD动量
  + 0.22 × max(oi_delta_ratio, 0)      // 持仓量增量 (正向)
  + 0.10 × max(funding_rate, 0)        // 正资金费率看空
  + 0.15 × max(-(depth_ratio - 1.0), 0) // 深度比率看空
  + 0.15 × max(-imbalance, 0)          // 买卖盘不平衡
  + liquidity_norm_factor × max(-liquidity_delta_norm, 0)  // 流动性减量
)
```

#### 4.1.3 动态权重AI优化 (DeepSeek)
当 `deepseek_weight_router.enabled = true` 时:
- 使用AI模型根据市场环境动态调整权重
- 默认权重作为本地规则回退
- 缓存TTL: 300秒

### 4.2 RANGE 模式评分公式

#### 4.2.1 做多评分 (long_score)
```
long_score_base = (
    0.55 × max(-imbalance, 0)          // 买盘过度做空
  + 0.35 × max(-cvd_momentum, 0)       // CVD动量反向
  + 0.10 × max(-(depth_ratio - 1.0), 0) // 深度比率看多
)
long_score = max(0, long_score_base - oi_penalty)
其中:
  oi_penalty = min(max(abs(oi_delta_ratio), 0), 1.0) × 0.20
```

#### 4.2.2 做空评分 (short_score)
```
short_score_base = (
    0.55 × max(imbalance, 0)           // 卖盘过度做多
  + 0.35 × max(cvd_momentum, 0)        // CVD动量同向
  + 0.10 × max(depth_ratio - 1.0, 0)   // 深度比率看空
)
short_score = max(0, short_score_base - oi_penalty)
其中:
  oi_penalty = min(max(abs(oi_delta_ratio), 0), 1.0) × 0.20
```

### 4.3 分数融合机制 (15m + 5m)

#### 4.3.1 融合公式
```
fused_long = (
    score_15m_weight × long_score_15m
  + score_5m_weight × long_score_5m
) × consistency_weight

fused_short = (
    score_15m_weight × short_score_15m
  + score_5m_weight × short_score_5m
) × consistency_weight
```

#### 4.3.2 参数配置
```json
{
  "score_15m_weight": 1.0,      // 15m权重
  "score_5m_weight": 0.0,       // 5m权重
  "consistency_window": 3       // 一致性检查窗口
}
```

#### 4.3.3 一致性加权
```
consistency_weight = 1.0 + 0.1 × (consistent_count - consistency_window + 1)
```
- 连续N根15m方向一致时增加权重
- 每多一根一致增加10%权重

## 六、特殊开仓机制

### 6.1 趋势捕获 (Trend Capture)

#### 6.1.1 触发条件
```json
{
  "trend_capture_enabled": true,
  "regime": "NO_TRADE",
  "trend_pending_side": "LONG" OR "SHORT",
  "trend_pending_score": 0.18,      // 待确认分数阈值
  "trend_capture_min_score": 0.22,   // 捕获分数阈值
  "trend_capture_min_gap": 0.05      // 最小分差
}
```

#### 6.1.2 趋势待确认评分
```
待确认条件:
  adx >= 18.0
  AND adx_slope >= 0.05
  AND 0.0008 <= atr_pct <= 0.02

做多评分:
  long_score = 0.0
  if (ema_spread > 0 AND ema_spread_expand >= 0.00015): long_score += 0.40
  if (ret_15m > 0): long_score += 0.20
  if (oi_delta_ratio > 0): long_score += 0.20
  if (ret_15m > 0 AND oi_delta_ratio > 0): long_score += 0.20

做空评分:
  short_score = 0.0
  if (ema_spread < 0 AND ema_spread_expand <= -0.00015): short_score += 0.40
  if (ret_15m < 0): short_score += 0.20
  if (oi_delta_ratio > 0): short_score += 0.20
  if (ret_15m < 0 AND oi_delta_ratio > 0): short_score += 0.20
```

#### 6.1.3 趋势捕获评分
```
做多信号要素:
  - 突破: close_5m >= hh_n AND cvd_momentum_5m > 0 (+0.10)
  - 回调延续: ema_fast_5m > ema_slow_5m AND ret_5m > 0 AND cvd_momentum_5m > 0 (+0.08)
  - CVD对齐: cvd_momentum_5m > 0 (+0.08)
  - OI对齐: oi_delta_ratio_5m > 0 AND ret_5m > 0 (+0.08)
  - 深度对齐: depth_ratio_5m >= 1.0 OR imbalance_5m > 0 (+0.05)
  - 微观确认: micro_delta > 0 AND microprice_bias > 0 (+0.04)

做空信号要素:
  - 突破: close_5m <= ll_n AND cvd_momentum_5m < 0 (+0.10)
  - 回调延续: ema_fast_5m < ema_slow_5m AND ret_5m < 0 AND cvd_momentum_5m < 0 (+0.08)
  - CVD对齐: cvd_momentum_5m < 0 (+0.08)
  - OI对齐: oi_delta_ratio_5m > 0 AND ret_5m < 0 (+0.08)
  - 深度对齐: depth_ratio_5m <= 1.0 OR imbalance_5m < 0 (+0.05)
  - 微观确认: micro_delta < 0 AND microprice_bias < 0 (+0.04)

微观结构惩罚:
  - trap_score > 0.65: -0.05
  - phantom_score > 0.65: -0.04
  - spread_z > 1.8: -0.04

确认要求:
  - 3m微观再加速: ret_3m > 0 AND micro_delta > 0 (做多)
  - 3m微观再加速: ret_3m < 0 AND micro_delta < 0 (做空)
  - 部分确认: 至少3个对齐要素满足 (扣0.03分)
```

#### 6.1.4 仓位乘数
```
trial_position_mult = 0.35  // 试探仓位35%
confirm_position_mult = 0.65 // 确认仓位65%
```

## 七、技术指标硬性过滤

### 7.1 MA10 + MACD 汇聚

#### 7.1.1 硬性阻止 (Hard Block)
```
阻止做多:
  - 价格在MA10下方: close_1h < ma10_1h
  - MACD做空触发: macd_line < macd_signal AND macd_hist < 0

阻止做空:
  - 价格在MA10上方: close_1h > ma10_1h
  - MACD做多触发: macd_line > macd_signal AND macd_hist > 0
```

#### 7.1.2 软性惩罚 (Soft Penalty)
```
做多惩罚:
  - 无MACD做多信号: -0.08
  - MACD早期信号(未金叉): -0.03
  - KDJ不支持: -0.04

做空惩罚:
  - 无MACD做空信号: -0.08
  - MACD早期信号(未死叉): -0.03
  - KDJ不支持: -0.04
```

### 7.2 MACD + KDJ 混合

#### 7.2.1 MACD趋势判定
```
MACD > 0:  看多趋势
MACD < 0:  看空趋势
MACD = 0:  中性区间 (±0.01内)
```

#### 7.2.2 KDJ超买超卖
```
J < 25:    超卖 (做多机会)
J > 75:    超买 (做空机会)
25 <= J <= 75: 中性
```

#### 7.2.3 组合信号
```
做多信号:
  - MACD > 0 (主趋势看多)
  - AND (J < 25 OR K >= D) (超卖或金叉)

做空信号:
  - MACD < 0 (主趋势看空)
  - AND (J > 75 OR K <= D) (超买或死叉)
```
