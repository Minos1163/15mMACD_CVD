## 1. 问题诊断

### 1.1 36H 未开仓根因分析

当前 V3.0 存在以下**过滤叠加问题**：

```
问题链路：

confluence_v2 检测
  ├─ require_macd_trigger = True      → 5m MACD 未金叉 → soft_penalty += 0.08
  ├─ not kdj_ok_long                  → KDJ 未对齐    → soft_penalty += 0.04
  └─ 叠加惩罚 = 0.12

TREND 模式 long_open_threshold = 0.07
典型 fused_long ≈ 0.10 ~ 0.15

最终: final_long_score = 0.12 - 0.12 = 0.00  ← 被惩罚清零
```

**KDJ 的核心问题**：
- KDJ 在震荡市（RANGE 模式）频繁钝化，K/D 长期交缠，kdj_ok 几乎永远不满足
- BTC 永续合约 5m/15m 的 KDJ 噪声极高，与资金流信号相关性弱
- KDJ 惩罚 0.04 叠加 MACD 惩罚 0.08 = **0.12 总惩罚**，几乎覆盖所有 TREND 模式的正常分数
- ADX 在 14.5~16 低趋势区间时，资金流分数本就偏低（0.08~0.14），再被扣 0.12 等于永远无法开仓

### 1.2 频率不足根因

| 指标 | 当前阈值 | 实际命中率 | 问题 |
|------|----------|-----------|------|
| KDJ 软惩罚 | -0.04 | 约 60% 时间触发 | 过度限制 |
| MACD_5m 金叉 | 要求 | 约 30% 时间满足 | 滞后 2-3 根 |
| 一致性权重 0.5 | 异向衰减 | 频繁触发 | 过于保守 |

## 2. 核心变更概述

### 2.1 变更对比表

| 模块 | V3.0 | V3.1 | 影响 |
|------|------|------|------|
| 辅助过滤指标 | KDJ（软惩罚 0.04） | **删除** | 减少误过滤 |
| 多周期 MACD | 无 | **MACD_1H + MACD_4H** | 加入方向确认 |
| 5m MACD 角色 | 入场触发（惩罚主力） | **执行确认（辅助）** | 降低阻拦力 |
| 开仓频率 | ~0.5 次/天 | **目标 2-3 次/天** | 频率提升 |
| 开仓阈值 TREND | 0.07 | **0.055** | 适当降低 |
| 软惩罚上限 | 无上限 | **≤ 0.06** | 防止过度惩罚 |
| 4H 背离拦截 | 无 | **HARD_BLOCK** | 风控加强 |
| 1H 柱子方向 | 无 | **方向加分 +0.03** | 顺势奖励 |

### 2.2 设计原则

> **宽进严出**：降低入场门槛，用更高维度的 MACD（1H/4H）做方向把控，而非用低维度 KDJ 做逐根 K 线拦截。

```
V3.0 哲学：细粒度过滤（KDJ + 5m MACD 双重卡口）→ 错误率低但频率极低
V3.1 哲学：大周期方向确认（1H/4H MACD）+ 小周期执行优化 → 频率提升同时保持方向胜率
```

### 4.2 MACD 多周期状态枚举

```python
# 新增枚举类
class MACDState(Enum):
    BULL_STRONG   = "bull_strong"    # 柱子为正且扩大
    BULL_WEAK     = "bull_weak"      # 柱子为正但收缩
    FLIP_TO_BULL  = "flip_to_bull"   # 柱子刚由负转正（★ 早期信号）
    BEAR_STRONG   = "bear_strong"    # 柱子为负且扩大（绝对值）
    BEAR_WEAK     = "bear_weak"      # 柱子为负但收缩（绝对值）
    FLIP_TO_BEAR  = "flip_to_bear"   # 柱子刚由正转负（★ 早期信号）
    NEUTRAL       = "neutral"        # 接近零轴

def classify_macd_state(hist: float, hist_prev: float, threshold=0.0) -> MACDState:
    if hist_prev <= threshold and hist > threshold:
        return MACDState.FLIP_TO_BULL      # 柱子翻红
    if hist_prev >= -threshold and hist < -threshold:
        return MACDState.FLIP_TO_BEAR      # 柱子翻绿
    if hist > threshold:
        return MACDState.BULL_STRONG if hist >= hist_prev else MACDState.BULL_WEAK
    if hist < -threshold:
        return MACDState.BEAR_STRONG if abs(hist) >= abs(hist_prev) else MACDState.BEAR_WEAK
    return MACDState.NEUTRAL
```

## 5. MACD 柱子背离检测

> **核心逻辑**：价格新高但 MACD 柱子未新高 = 顶背离（做空信号 / 拦截做多）；价格新低但 MACD 柱子未新低 = 底背离（做多信号 / 拦截做空）

### 5.1 背离检测实现

```python
def detect_macd_divergence(
    price_series: list[float],    # 最近 N 根收盘价
    hist_series: list[float],     # 最近 N 根 MACD 柱子
    lookback: int = 5             # 回看根数
) -> dict:
    """
    返回:
      bearish_div: bool  顶背离（价格新高 + 柱子未新高）→ 警示做多 / 利空
      bullish_div: bool  底背离（价格新低 + 柱子未新低）→ 警示做空 / 利多
    """
    recent_prices = price_series[-lookback:]
    recent_hists  = hist_series[-lookback:]

    price_new_high = recent_prices[-1] >= max(recent_prices[:-1])
    price_new_low  = recent_prices[-1] <= min(recent_prices[:-1])
    hist_new_high  = recent_hists[-1]  >= max(recent_hists[:-1])
    hist_new_low   = recent_hists[-1]  <= min(recent_hists[:-1])

    bearish_divergence = price_new_high and (not hist_new_high)   # 顶背离
    bullish_divergence = price_new_low  and (not hist_new_low)    # 底背离

    return {
        "bearish_div": bearish_divergence,
        "bullish_div": bullish_divergence,
    }
```

### 5.2 背离信号在风控中的应用

#### 4H 背离 → HARD_BLOCK（最高级别拦截）

```python
# 4H 顶背离：强制拦截做多开仓
div_4h = detect_macd_divergence(price_4h_series, macd_4h_hist_series)

if div_4h["bearish_div"]:
    # 4H 顶背离：HARD_BLOCK 做多
    hard_block_long = True
    block_reason = "4H_MACD_BEARISH_DIVERGENCE"

if div_4h["bullish_div"]:
    # 4H 底背离：HARD_BLOCK 做空
    hard_block_short = True
    block_reason = "4H_MACD_BULLISH_DIVERGENCE"
```

#### 1H 背离 → SOFT_PENALTY（软性惩罚）

```python
div_1h = detect_macd_divergence(price_1h_series, macd_1h_hist_series)

if div_1h["bearish_div"]:
    # 1H 顶背离：软性惩罚做多
    soft_penalty_long += 0.05
    divergence_warn_long = True

if div_1h["bullish_div"]:
    # 1H 底背离：软性惩罚做空
    soft_penalty_short += 0.05
    divergence_warn_short = True
```

## 6. MACD 柱子翻色早期信号

> **核心逻辑**：柱子从负转正（翻红）比金叉提前 2-3 根 K 线，是更早的入场信号

### 6.1 翻色信号定义

```python
# 1H 柱子翻色信号
macd_1h_flip_bull = (macd_1h_hist_prev <= 0) and (macd_1h_hist > 0)   # 1H 柱子翻红
macd_1h_flip_bear = (macd_1h_hist_prev >= 0) and (macd_1h_hist < 0)   # 1H 柱子翻绿

# 4H 柱子翻色信号（更强，代表更大级别趋势转换）
macd_4h_flip_bull = (macd_4h_hist_prev <= 0) and (macd_4h_hist > 0)   # 4H 柱子翻红
macd_4h_flip_bear = (macd_4h_hist_prev >= 0) and (macd_4h_hist < 0)   # 4H 柱子翻绿
```

### 6.2 翻色信号加分逻辑

```python
# ---- 做多方向 ----
# 1H 柱子翻红：方向加分（早期入场奖励）
if macd_1h_flip_bull:
    long_score_bonus += 0.03          # 奖励早期信号
    macd_1h_direction = "BULL"

# 4H 柱子翻红：更强的方向确认，额外加分
if macd_4h_flip_bull:
    long_score_bonus += 0.04          # 4H 级别转换，更大奖励
    macd_4h_direction = "BULL"

# 1H 柱子持续为正（非翻色但方向对齐）：小幅加分
elif macd_1h_state in (MACDState.BULL_STRONG, MACDState.BULL_WEAK):
    long_score_bonus += 0.015
    macd_1h_direction = "BULL"

# ---- 做空方向（对称逻辑）----
if macd_1h_flip_bear:
    short_score_bonus += 0.03
    macd_1h_direction = "BEAR"

if macd_4h_flip_bear:
    short_score_bonus += 0.04
    macd_4h_direction = "BEAR"

elif macd_1h_state in (MACDState.BEAR_STRONG, MACDState.BEAR_WEAK):
    short_score_bonus += 0.015
    macd_1h_direction = "BEAR"
```

### 6.3 大周期方向不一致惩罚

```python
# 1H/4H 方向与开仓方向相反时，施加惩罚（但不 HARD_BLOCK）
if macd_4h_direction == "BEAR" and long_score > short_score:
    # 4H 偏空但要做多 → 强惩罚
    soft_penalty_long += 0.04

if macd_1h_direction == "BEAR" and long_score > short_score:
    # 1H 偏空但要做多 → 轻惩罚
    soft_penalty_long += 0.02

# 做空方向对称
if macd_4h_direction == "BULL" and short_score > long_score:
    soft_penalty_short += 0.04

if macd_1h_direction == "BULL" and short_score > long_score:
    soft_penalty_short += 0.02
```

## 7. 开仓频率优化

### 7.1 阈值调整

当前 TREND 模式阈值 0.07 在实际分数分布下触发率过低。调整如下：

```json
// TREND 模式
"long_open_threshold":  0.055,   // ← 从 0.07 降至 0.055
"short_open_threshold": 0.050,   // ← 从 0.06 降至 0.050

// NO_TRADE 趋势捕获
"trend_capture_min_score": 0.18, // ← 从 0.22 降至 0.18
"trend_capture_min_gap":   0.04, // ← 从 0.05 降至 0.04

// RANGE 模式（不变，已有足够频率）
"long_open_threshold":  0.48,
"short_open_threshold": 0.40
```

> ⚠️ **注意**：RANGE 模式阈值保持不变，因为 RANGE 模式频率问题不严重，且区间极端反转要求严格确认。

### 7.2 软惩罚上限保护

新增全局软惩罚上限，防止多重惩罚叠加导致永远无法开仓：

```python
# 软惩罚上限：最大不超过阈值的 80%
SOFT_PENALTY_MAX_LONG  = long_open_threshold  * 0.8   # TREND: 0.055 × 0.8 = 0.044
SOFT_PENALTY_MAX_SHORT = short_open_threshold * 0.8   # TREND: 0.050 × 0.8 = 0.040

soft_penalty_long  = min(soft_penalty_long,  SOFT_PENALTY_MAX_LONG)
soft_penalty_short = min(soft_penalty_short, SOFT_PENALTY_MAX_SHORT)
```

### 7.3 MACD_5m 角色降级

原 5m MACD 作为"入场触发"（不满足惩罚 0.08），调整为"执行优化"（不满足惩罚降为 0.04）：

```python
# V3.0（原来）
if not macd_trigger_long:
    soft_penalty_long += 0.08    # ← 过重

# V3.1（新）
if not macd_trigger_long:
    if macd_early_long:           # 柱子翻红算"早期满足"
        soft_penalty_long += 0.02
    else:
        soft_penalty_long += 0.04  # ← 从 0.08 降至 0.04
```

## 8. 新评分公式重构

### 8.1 V3.1 TREND 模式完整评分流程

```
┌─────────────────────────────────────────────────────────┐
│              多周期 MACD 方向预判（新增）                 │
│  4H MACD 柱子状态  →  direction_4h: BULL / BEAR / NEUTRAL│

│  4H/1H 背离检测    →  div_4h / div_1h                   │

                    │ 方向过滤（HARD_BLOCK 4H背离）
                    ↓
┌─────────────────────────────────────────────────────────┐
│              资金流基础评分（保持不变）                    │
│  long_score / short_score = 原有 CVD + OI + 深度等      │
└───────────────────┬─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│              MACD 多周期奖励 / 惩罚（新增）               │
│  +0.04  4H 柱子翻色（同向）                              │
│  +0.03  1H 柱子翻色（同向）                              │
│  +0.015 1H 柱子持续同向                                  │
│  -0.04  4H 方向相反惩罚                                  │
│  -0.02  1H 方向相反惩罚                                  │
│  -0.05  1H 背离软惩罚                                    │
└───────────────────┬─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│              5m MACD 执行惩罚（降级，原为主力）           │
│  未触发 + 无早期信号: -0.04（← 从 -0.08 降低）           │
│  柱子翻色早期信号:    -0.02                              │
│  KDJ 惩罚:           删除                               │
│  软惩罚上限:          ≤ threshold × 0.8                 │
└───────────────────┬─────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────┐
│              开仓判定                                     │
│  final_score ≥ threshold (0.055) → 开仓                  │
└─────────────────────────────────────────────────────────┘
```

### 8.2 V3.1 分数示例对比

**场景**：TREND 做多，5m MACD 未金叉，1H 柱子翻红，4H 柱子持续为正

```
  净调整:     +0.03 - 0.02 = +0.01

```

## 9. 风控逻辑重构

### 9.1 新增 MACD 多周期风控层

在 `_resolve_entry_mode` 中新增以下检查层级：

```
风控优先级（从高到低）：

LEVEL 1 [HARD_BLOCK]  4H MACD 背离（与开仓方向相反的顶/底背离）
LEVEL 2 [HARD_BLOCK]  原有 block_reverse_macd 硬拦截
LEVEL 3 [SOFT -0.05]  1H MACD 背离（同方向发散）
LEVEL 4 [SOFT -0.04]  4H 方向不一致惩罚
LEVEL 5 [SOFT -0.02]  1H 方向不一致惩罚
LEVEL 6 [SOFT -0.04]  5m MACD 未触发（降级后）
LEVEL 7 [上限保护]    soft_penalty ≤ threshold × 0.8
```

### 9.2 背离 + 反转组合信号（高胜率场景）

当 **4H 顶背离 + 1H 柱子翻绿** 同时出现时，触发强制做空信号加成：

```python
# 高胜率做空组合
if (div_4h["bearish_div"] and macd_1h_flip_bear):
    # 背离确认 + 柱子翻色：强力做空信号
    if short_score > long_score:
        short_score_bonus += 0.06     # 额外奖励高胜率组合
        high_confidence_short = True   # 标记高置信度，可适当提高仓位

# 高胜率做多组合（底背离 + 1H 柱子翻红）
if (div_4h["bullish_div"] and macd_1h_flip_bull):
    if long_score > short_score:
        long_score_bonus += 0.06
        high_confidence_long = True
```

### 9.3 止损优化（结合大周期 MACD）

```python
# 当 4H 和 1H 均对齐时，适当放宽止损（趋势更确定）
if macd_4h_direction == macd_1h_direction == intended_direction:
    stop_loss_pct *= 0.90   # 止损缩小 10%（更紧，更确定）

# 当 4H 对齐但 1H 背离时，适当收紧止损
if macd_4h_direction == intended_direction and div_1h_against:
    stop_loss_pct *= 1.10   # 止损扩大 10%（更宽，需要更多容错空间）
```
