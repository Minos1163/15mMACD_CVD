# 交易执行引擎设计

## 7.1 模块职责

### 7.1.1 核心功能

- 订单下单
- 订单撤单
- 订单修改
- 持仓同步
- 异常处理
- 断线恢复

### 7.1.2 设计原则

- 幂等性：重复请求不会产生副作用
- 容错性：网络异常自动重试
- 可追溯：所有操作记录日志

## 7.2 订单类型

### 7.2.1 支持的订单类型

| 类型 | 说明 | 使用场景 |
|-----|------|---------|
| limit | 限价单 | 网格挂单 |
| market | 市价单 | 紧急平仓/止损 |
| stop_loss | 止损单 | 风险控制 |

### 7.2.2 订单方向

| 方向 | 说明 |
|-----|------|
| buy | 买入 |
| sell | 卖出 |

## 7.3 执行逻辑

### 7.3.1 网格订单生成流程

1. 根据策略生成目标网格价格列表
2. 检查当前已有订单
3. 计算需要新增/撤销的订单
4. 批量提交订单请求
5. 等待订单成交或超时

### 7.3.2 订单维护流程

1. 定期检查订单状态
2. 更新成交统计
3. 触发补单逻辑
4. 清理超时/失败订单

### 7.3.3 成交处理流程

1. 接收成交推送
2. 更新持仓信息
3. 计算收益情况
4. 触发补单（如果需要）
5. 更新策略状态

## 7.4 异常处理

### 7.4.1 网络异常

- 自动重试（默认 3 次）
- 指数退避策略
- 记录失败日志

### 7.4.2 交易所返回异常

| 错误码 | 处理方式 |
|-------|---------|
| -1000 | 网络异常，重试 |
| -2015 | 订单不存在，跳过 |
| -2013 | 余额不足，等待 |
| -1111 | 精度错误，调整价格 |

### 7.4.3 部分成交

- 更新剩余数量
- 保持挂单状态
- 记录成交详情

## 7.5 状态同步

### 7.5.1 同步周期

- 订单状态：实时推送 + 定期轮询
- 持仓信息：每次成交后
- 账户权益：每次下单前

### 7.5.2 断线恢复

1. 重新连接交易所
2. 查询当前订单状态
3. 查询当前持仓
4. 对比差异
5. 补齐缺失订单
6. 恢复策略执行

## 7.6 性能优化

### 7.6.1 批量操作

- 合并同类订单请求
- 减少 API 调用次数

### 7.6.2 缓存策略

- 缓存订单簿信息
- 减少高频查询

### 7.6.3 限流控制

- 遵守交易所 API 限制
- 预留缓冲容量
